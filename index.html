A production-grade, three-tier weather forecast platform showcasing a complete end-to-end DevOps workflow — automated CI/CD, containerized microservices, Infrastructure as Code on AWS, real-time monitoring hooks, and scheduled email alerting. Zero manual steps from git push to production.

  • Implemented containerized microservices architecture: Designed a three-tier application (FastAPI backend · Vanilla JS frontend · PostgreSQL) using Docker & Docker Compose, enabling consistent, reproducible deployments across dev/staging/production environments.
  • Automated CI/CD pipeline end-to-end: Built with GitHub Actions — automated Flake8 linting, pytest unit testing, multi-stage Docker image build & push to Docker Hub, and SSH-triggered rolling deployment on AWS EC2. Reduced deployment time from ~20 minutes (manual) to under 2 minutes (fully automated).
  • Provisioned Infrastructure as Code with Terraform: Declaratively managed all AWS resources (EC2, Security Groups, Elastic IP, IAM roles) via Terraform — eliminating manual console operations and enabling one-command terraform apply infrastructure provisioning and teardown.
  • Configured Nginx Reverse Proxy: Set up Nginx as the single ingress point — routing HTTP traffic between the frontend static assets and FastAPI backend, with proper upstream proxy headers, improving security by eliminating direct backend exposure.
  • Built automated weather email notification service: Implemented a scheduled Python cron service integrating OpenWeatherMap API — delivering rich HTML weather reports (UV index, AQI, hourly forecasts, health advisories) to registered users, with 100% automated scheduling via cron.
